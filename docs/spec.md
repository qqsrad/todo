**バージョン**: 1.7  
**作成日**: 2026-04-07

## 1. システム概要

### 1.1 目的

ローカル環境で動作するTodo管理アプリケーション。Node.js + SQLiteで構成し、ブラウザまたはElectronデスクトップウィンドウから利用する。

### 1.2 技術スタック

| レイヤー | 技術 |
|---|---|
| バックエンド | Node.js + Express |
| データベース | SQLite（better-sqlite3） |
| フロントエンド | Vanilla JS + HTML + CSS |
| デスクトップシェル | Electron |
| Markdownレンダリング | marked.js（CDN） |

### 1.3 ディレクトリ構成

```
todo-app/
├── _run-electron.bat
├── _run-electron.vbs
├── server.js
├── electron/
│   └── main.js
├── db.js
├── routes/
│   ├── todos.js
│   ├── categories.js
│   └── export.js
├── public/
│   ├── favicon.ico      # Web/Electron共通アイコン
│   ├── app-icon.png     # アイコン元画像
│   ├── index.html       # 一覧画面
│   ├── app.js
│   └── style.css
└── package.json
```

## 3. 定数・マスタ定義

### 3.1 ステータス

| 値 | 表示ラベル | 完了扱い |
|---|---|---|
| undecided | 未決定 | × |
| todo | 未着手 | × |
| in_progress | 着手済み | × |
| requested | 依頼中 | × |
| almost | あと少し | × |
| done | 完了 | ○ |
| hold | 保留 | × |

### 3.2 優先度

| 値 | 表示ラベル | 備考 |
|---|---|---|
| -1 | 非表示 | 一覧に表示しない |
| 0 | 低優先 | |
| 1 | 普通 | デフォルト |
| 2 | 重要 | |
| 3 | 超重要 | |

### 3.3 ソース

初期値として以下を用意（自由入力も可）：

- メール
- チャット
- Teams
- 口頭
- その他

### 3.4 時刻種別

| 値 | 表示ラベル | 時刻 |
|---|---|---|
| none | 指定なし | NULL |
| start | 始業 | 08:50 |
| custom | 時刻指定 | ユーザー入力 |
| end | 定時内 | 17:20 |
| eod | EoD | 23:59 |

`始業` / `定時内` / `EoD` など既定時刻を持つ種別を選んだ後に `時刻指定` へ切り替えた場合、時刻入力欄には直前に選んだ種別の時刻を初期値として表示する。`指定なし` から切り替えた場合は空欄にする。

## 4. 締め切り日のプリセット変換

### 4.1 入力時（プリセット→日付変換）

登録・編集画面で以下のプリセットから選択すると、対応する日付に変換して保存する。

| プリセット | 変換後の日付 |
|---|---|
| 今日中 | 当日 |
| 今週中 | 当週の金曜日 |
| 来週中 | 翌週の日曜日 |
| 今月中 | 当月末日 |
| 今Q中 | 現在の四半期末日（Q1:6/30, Q2:9/30, Q3:12/31, Q4:3/31） |
| 今期中 | 現在の半期末日（上期:9/30, 下期:3/31） |
| 今年度中 | 当年度の3月末日 |

※ 四半期・期は日本の会計年度（4月始まり）に準拠。
※ 「今日中」を選んだ場合、時刻種別は `定時内` を自動選択する。

### 4.2 表示時（日付→プリセット逆変換）

時刻指定なし（`deadline_time_type = none`）の場合のみ適用。優先度は上から順に最初にマッチしたものを表示する。

1. 今日中（当日）
2. 今週中（当週金曜以内）
3. 来週中（翌週日曜以内）
4. 今月中（当月末以内）
5. 今Q中（四半期末以内）
6. 今期中（半期末以内）
7. 今年度中（年度末以内）
8. それ以外 → 日付をそのまま表示（YYYY/MM/DD）

## 5. 一覧画面

### 5.1 表示レイアウト

Windowsエクスプローラーの「詳細表示」に近い形式のテーブル。

表示カラム（左から順）：

| カラム | 内容 | ソート | インライン編集 |
|---|---|---|---|
| 完了ボタン | ワンクリック完了 | × | × |
| タイトル | タイトル | ○ | ○（テキスト入力） |
| 締め切り | 日付（曜日付き、必要に応じてプリセット逆変換表示）+ 時刻 | ○ | ○（日付ピッカー） |
| ソース | ソース | ○ | ○（セレクト） |
| 振出人 | 振出人 | ○ | ○（テキスト入力） |
| 依頼先 | 依頼先 | ○ | ○（テキスト入力） |
| カテゴリ | カテゴリ名 | ○ | ○（セレクト） |
| 優先度 | ラベル | ○ | ○（セレクト） |
| ステータス | ラベル | ○ | ○（セレクト） |
| 登録日時 | YYYY/MM/DD HH:MM | ○ | × |
| 操作 | 編集・削除ボタン | × | × |

### 5.2 視覚的ハイライト

#### 締め切りによる行ハイライト

| 条件 | 色 |
|---|---|
| 期限超過（過去） | 赤系背景 |
| 今日中 | オレンジ系背景 |
| 3日以内 | 黄色系背景 |
| それ以外 | デフォルト |

#### 優先度による文字色

| 優先度 | 色 |
|---|---|
| 3（超重要） | 赤 + 太字 |
| 2（重要） | オレンジ |
| 1（普通） | デフォルト |
| 0（低優先） | グレー |

#### 完了済み行

- 文字色をグレーアウト
- タイトルに取り消し線

#### ステータスバッジ

各ステータスに色付きバッジを表示。

### 5.3 完了ボタン

- 各行の先頭にチェックボタンを配置
- クリックするとステータスを `done`（完了）に変更
- 完了済み行の場合は未着手（`todo`）に戻す

### 5.4 完了フィルタ

画面上部にトグルで表示対象を切り替え：

- **未完了のみ**（デフォルト）：`done` 以外を表示
- **完了済みのみ**：`done` のみを表示
- **すべて**：全件表示

### 5.5 グループ化

3種類のグループ化モードを選択できる：

#### モード1：優先度でグループ化（各グループ内は期限順）

グループ順：3（超重要）→ 2（重要）→ 1（普通）→ 0（低優先）  
優先度ラベルをグループヘッダーとして表示。

#### モード2：期限でグループ化（各グループ内は優先度順）

グループ順：

1. 期限超過
2. 今日中
3. 3日以内
4. 7日以内
5. 2週間以内
6. 1ヶ月以内
7. それ以上
8. 期限なし

各グループ内は優先度の降順（3→0）で表示。

#### モード3：グループなし

デフォルトソート：未完了・期限が近い順（期限昇順、期限なしは末尾）。

### 5.6 グループの折りたたみ

- 各グループヘッダーをクリックで展開/折りたたみ切り替え
- 折りたたみ状態はセッション中記憶

### 5.7 ソート

- カラムヘッダークリックで昇順/降順切り替え
- ソート中のカラムは矢印アイコンで表示

### 5.8 検索・フィルタ

#### キーワード検索

- 検索対象：タイトル + カテゴリ名 + 振出人 + 依頼先（デフォルト）
- オプション切り替えでメモも検索対象に追加可能

#### フィルタ

以下の条件を複合で指定可能：

| フィルタ項目 | 入力方法 |
|---|---|
| カテゴリ | セレクト（複数選択可） |
| ソース | セレクト（複数選択可） |
| 優先度 | チェックボックス |
| ステータス | チェックボックス |
| 期限範囲（プリセット） | セレクト（今日/3日/7日/2週間/1ヶ月/今月中 等） |
| 期限範囲（日付指定） | From〜To の日付入力 |

## 6. 登録・編集画面

### 6.1 フォーム項目

| 項目 | 入力方法 | 必須 | デフォルト |
|---|---|---|---|
| タイトル | テキスト入力 | ○ | |
| 締め切り日 | カレンダー or プリセット選択 | × | |
| 時刻種別 | セレクト | × | 前回選択を記憶 |
| 時刻（時刻指定時のみ） | HH:MM 入力 | × | |
| ソース | セレクト（自由入力も可） | × | |
| 振出人 | テキスト入力（過去入力値から選択可） | × | |
| 依頼先 | テキスト入力（過去入力値から選択可） | × | |
| カテゴリ | セレクト | × | 未分類 |
| 優先度 | セレクト（-1〜3） | ○ | 1（普通） |
| ステータス | セレクト | ○ | 未着手 |
| メモ | Markdownエディタ（プレビュー切替可） | × | |

### 6.2 締め切り日の入力

- カレンダー（日付ピッカー）と、プリセットボタン群の両方を用意
- プリセット選択時は変換後の日付をカレンダーに反映して表示
- 締め切り日の表示は曜日付き（例: `2026/04/07 (火)`）とする
- 編集画面では締め切り日の表示テキスト自体を編集入口として使う
- 締め切り日の表示テキスト押下時は、その表示位置を日付入力欄に置き換えてカレンダーを表示する
- カレンダーボタンは常時表示し、押下時も同様に日付入力欄へ切り替える
- 日付選択またはプリセット選択の完了後は日付入力とプリセットを非表示に戻す
- プリセットボタン：今日中 / 今週中 / 来週中 / 今月中 / 今Q中 / 今期中 / 今年度中 / クリア

### 6.2.1 フォーム配置

- 締め切り日と時刻は同一行で表示する
- ソース・振出人・依頼先は同一行で表示する
- カテゴリ・優先度・ステータスは同一行で表示する

### 6.2.2 振出人・依頼先の候補表示

- 振出人・依頼先はテキスト入力とする
- 既存Todoで一度保存した値は次回以降の入力候補として選択できる
- 候補はそれぞれ振出人列、依頼先列の保存済み値から重複除去して表示する

### 6.3 メモ欄

- Markdownで入力
- 「編集」「プレビュー」タブで切り替え。新規登録・編集画面を開いた直後はプレビューを表示する
- プレビューはmarked.jsでレンダリング
- Electronデスクトップ起動時、プレビュー内の外部HTTP/HTTPSリンクはアプリ内遷移せず既定ブラウザで開く

### 6.4 自動入力

- **登録日時**（created_at）：新規登録時にサーバー側で現在日時を設定
- **最終更新日時**（updated_at）：保存時にサーバー側で更新

## 10. 起動方法

```bash
# 初回
npm install
node server.js

# ブラウザでアクセス
http://localhost:3000
```

PowerShell では実行ポリシーで `npm` がブロックされる場合があるため、必要に応じて `npm.cmd` を使う。

```powershell
# PowerShell例
npm.cmd install
npm.cmd run dev
```

`server.js` は `--host` と `--port` の起動引数、または `HOST` / `PORT` 環境変数を受け付ける。  
例：`node server.js --host 0.0.0.0 --port 6040`

### Electronデスクトップ起動

Electron シェル経由で、ローカルサーバーの起動完了後に専用ウィンドウを自動表示できる。

```powershell
npm.cmd install
npm.cmd run desktop
```

ダブルクリック起動用に `_run-electron.bat` を用意する。  
Electron 起動時はローカルにインストール済みの `node.exe` で `server.js` を起動し、`127.0.0.1` の空きポートで待ち受けた URL をアプリ内ウィンドウで開く。

見えるウィンドウを Electron のみ 1 つにしたい場合は `_run-electron.vbs` を使う。  
`_run-electron.bat` は保守用の入口として残し、通常利用では VBScript ランチャーから黒いコンソールを表示せず起動できる。

### 10.1 アプリアイコン

- Web 版の favicon と Electron タイトルバーアイコンは `public/favicon.ico` を共通利用する
- アイコンの元画像は `public/app-icon.png` とし、Todo のチェックリストをモチーフにしたデザインを採用する

### Windowsサービス化（オプション）

pm2やnode-windowsを利用してWindowsサービスとして登録可能。

```bash
npm install -g pm2
pm2 start server.js --name todo-app
pm2 save
pm2 startup
```
