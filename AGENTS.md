# 必須ルール

## 環境

- Use PowerShell
- Use encoding utf-8
- 必ずUTF-8 (BOMなし)で読み書きする
- 文字コードを絶対に変えない
- 日本語を含むファイルはCP932ではなくUTF-8として扱う
- npm run buildによるビルド確認は PowerShell の実行ポリシーで npm がブロックされるため、npm.cmd で実行する

## データ / バックエンド

- SQLite を使うローカル複数サービス構成なので、接続やロックを長時間保持しない。起動時・同期時・移行時も不要な排他を避ける
- エラーや競合は 500 で隠さず、利用者や呼び出し側が次の行動を判断できる粒度で返す

## UI状態 / 非同期

- 非同期取得UIは `loading / error / empty / success` を必ず明示し、失敗時に無表示（`return null`）を避ける
- エラーメッセージは利用者に見える形で表示し、必要に応じて `role="alert"` や `role="status"` を付与する
- 日時表示・日付集計・期間判定は、画面仕様として日本時間が前提なら JST を明示的に使い、実行環境のローカルタイムに暗黙依存しない

## アーキテクチャ / 実装方針

- 入力処理は可能な限り `form` + `FormData` で扱い、不要な `useState` を増やさない
- 画面の見た目再現よりも、意味的なHTML構造と保守しやすい状態設計を優先する

## UI名称 / ドキュメント

- 実装で仕様や挙動を変えた場合は、関連する docs も同じ変更で更新し、実装だけ先行して仕様書を放置しない
